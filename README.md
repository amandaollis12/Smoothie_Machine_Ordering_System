# Smoothie Machine Ordering System  
*A Theory of Computation Project using Automata, Regular Languages, and State Design*

## Overview  
This project models a **Smoothie Machine Ordering System** using concepts from **formal languages and automata theory**. The system simulates a customer ordering process by defining states, transitions, valid inputs, and acceptance conditions.

---

## Project Goals  
- Represent the smoothie ordering workflow using **Deterministic Finite Automata (DFA)**  
- Show alternate customer paths using a **Non-Deterministic Finite Automaton (NFA)**
- Modeled the ordering process with a with a **Context Free Grammar**
- Describe valid order sequences using a **Regular Expression**  
- Define a complete **alphabet** of valid user inputs  
- Create **state diagrams** with at least 5 states  
- Include a **reset state** and a **reject/dead state** for invalid inputs
